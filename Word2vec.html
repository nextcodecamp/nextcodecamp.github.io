Word2Vec เป็นโมเดลที่พัฒนาขึ้นด้วยภาษาไพธอนที่สนับสนุนการประมวลผลภาษาธรรมชาติ โดย Tomas Mikolov นักคอมพิวเตอร์สัญชาติเชคฯ เป็นผู้พัฒนาขณะที่ทำงานกับ Google
Word2vec จัดการข้อความหรือ text ด้วยวิธีการที่เรียกว่า
Word embeddings ซึ่งเป็นวิธีที่แสดงข้อความในรูปแบบของตัวเลขในเวคเตอร์ อาจเรียก Word2Vec ว่าเป็นวิธีการในการนำเสนอข้อความแบบเวคเตอร์ที่แสดงด้วยตัวเลข ซึ่งสามารถนำไปพิจารณาวิเคราะห์ความเหมือนของข้อความ (text similarity) หรือนำไปประยุกต์ใช้ในระบบแนะนำ (recommendation systems) และการวิเคราะห์อารมณ์หรือความรู้สึก (sentiment analysis) ได้

Vector space model สามารถแทนค่าข้อความในเอกสารและคำสั่งสอบถาม ดังสมการตัวอย่างต่อไปนี้ เมื่อ dj คือเอกสารที่ j ซึ่งมีคำในเอกสารในตำแหน่งที่ 1 ไปจนถึง t โดย  W1 ไปจนถึงคำที่ Wt คือค่าน้ำหนักที่คำนวณจาก TF/IDF หรือความถี่ของคำที่ปรากฎในเอกสาร 
dj=(w1,j,w2,j,…,wt,j)
สมมติตัวอย่างเอกสารมีข้อความคือ 
"ฉันไปเที่ยวที่งานกาชาดกับเพื่อนๆคือ เอ แอน บี ฉันสนุกมาก" ถ้าคำคือ ฉัน กาชาด เพื่อน เอ แอน บี TF/IDF ของคำว่า ฉัน จะมีค่าน้ำหนักสูงสุด ในขณะที่คำอื่นๆ ปรากฎเพียงหนึ่งครั้งในเอกสาร

TF/IDF คือวิธีการในการกำหนดน้ำหนักของคำที่ปรากฎในเอกสาร ซึ่งหลักการนี้ใช้ใน Information retrieval และคำนวณจากค่า TF และ IDF

TF - Term frequency คือค่าสัดส่วนความถี่ของคำที่ปรากฎในเอกสาร ซึ่งพิจารณาจากคำทั้งหมด เช่น หากพิจารณา TF ของคำว่า ฉัน จากข้อความ "ฉันไปเที่ยวที่งานกาชาดกับเพื่อนๆคือ เอ แอน บี ฉันสนุกมาก" ซึ่งคำทั้งหมดคือ ฉันปรากฎ 2 ครั้ง ส่วนคำที่เหลือคือ กาชาด เพื่อน เอ แอน และ บีปรากฎเพียงแค่ 1 ครั้ง ดังนั้นค่า TF ของคำว่าฉัน คือ 2/7 หรือ 0.285   ส่วนคำอื่นๆจะมี TF 1/7 หรือ 0.143
อย่างไรก็ตามวิธีการนี้มักกำจัดคำที่ไม่ใช่สาระสำคัญของเอกสารออกไป เพราะคำบางคำมักปรากฎบ่อยๆในเอกสาร เช่น ในเอกสารภาษาอังกฤษ the, a, an จะไม่ถูกนำมาพิจารณา ส่วนในภาษาไทย คำว่า การ  ฉัน เรา บางครั้งก็ไม่ใช่สาระสำคัญของเอกสารที่ควรนำมาพิจารณา ซึ่งในการวิเคราห์ขึ้นอยู่กับเรื่องหรือโดเมนที่จะต้องกำหนดคำเหล่านี้ต่อไป 

IDF - Inverse document frequency ค่า log ของจำนวนเอกสารทั้งหมดหารด้วยจำนวนเอกสารที่มีคำนั้นปรากฎ ค่า IDF เป็นตัวบ่งชี้ความสำคัญของคำจากกลุ่มเอกสารทั้งหมดที่นำมาวิเคราะห์ โดยให้น้ำหนักไปที่คำที่พบได้น้อยในกลุ่มเอกสาร  เช่น จาก 10 เอกสารมีคำว่า ฉัน ปรากฎ 8 เอกสาร ดังนั้น IDF มีค่าเป็น log (10/8) หรือ 0.096  ซึ่งเข้าใกล้ 0 จะแสดงให้เห็นว่าคำคำนี้อาจไม่ใช่สาระสำคัญของกลุ่มเอกสาร  แต่คำที่ทำให้ IDF เข้าใกล้ 1 จะเป็นคำที่มีความสำคัญมากกว่า จะเห็นว่า IDF พยายามหักล้าง TF ในการกำหนดความสำคัญของคำจากกลุ่มเอกสารนั่นเอง ซึ่ง TF ให้ความสำคัญกับคำที่พบบ่อย ส่วน IDF ให้ความสำคัญกับคำที่พบได้น้อยกว่า

 TF/IDF จะคำนวณจาก ค่า TF คูณด้วย IDF  จากตัวอย่างคำว่า ฉัน จะมีค่า TF/IDF = 0.285 x 0.096 = 0.027 

Word embedding

ภาพจาก https://www.tensorflow.org/text/guide/word_embeddings

Word embedding คือวิธีการแสดงข้อความหรือเท็กซ์ให้อยู่ในรูปแบบของเวคเตอร์ที่แสดงค่าข้อมูลด้วยตัวเลข 
Google แนะนำ One-hot encoding เป็นวิธีพื้นฐานในการแสดงคำ (Vocabulary) ที่ปรากฎในข้อความ เช่น จากภาพ Vocabulary แสดงในคอลัมภ์ของเวคเตอร์ และคำแต่ละคำจากเอกสารหรือข้อความจะสัมพันธ์กับเวคเตอร์ เช่น กำหนด Dimensions ของเวตเคอร์แสดงคำศัพท์ คือ tonight I, you, eating, home
                           tonight  I     you     eating   home     
ประโยคคือ Tonight, I am waiting for you for eating together at home.
tonight = [ 1, 0, 0, 0, 0]
I = [0,1,0,0,0]
am=[0,0,0,0,0]
waiting=[0,0,0,0,0]
for=[0,0,0,0,0]
you=[0,0,1,0,0]
eating=[0,0,0,1,0]
together=[0,0,0,0,0]
at=[0,0,0,0,0]
home=[0,0,0,0,1] 

ข้อเสียของ One-hot encoding คือจะมีเวคเตอร์ที่มีค่า 0 อยู่เป็นจำนวนมาก ซึ่งอาจทำให้เสียหน่วยความจำมาก เราเรียกเวคเตอร์ลักษณะนี้ว่า Sparse vector 
อีกเทคนิคหนึ่งที่สามารถทำ encoding ได้คือ การระบุหมายเลขของคำทีพบเช่น 
vector = [1,2,6,7,10] 
แต่วิธีนี้ก็ยังไม่สามารถแสดงให้เห็นความสัมพันธ์ของคำได้

Word embedding ใช้เทคนิคในการวิเคราะห์ความสัมพันธ์ของคำ โดยพิจารณาบริบทของคำ (ตำแหน่งของคำและคำอื่นๆข้างเคียง)
เวคเตอร์แสดงค่าน้ำหนักที่ได้จากการเรียนรู้โดยใช้ฐานข้อมูลคำหรือคอร์ปัส  Neural network นำไปใช้ในการเรียนรู้เพื่อหาค่าน้ำหนักของคำ ซึ่งค่าน้ำหนักของคำในเวคเตอร์แสดงความสัมพันธ์เชิงความหมาย (พิจารณาจากบริบทของคำ) ซึ่งได้จากการเรียนรู้เวคเตอร์อาจมีขนาด 4-dimension ไปจนถึง 1024-dimension ตัวอย่าง (ดูภาพซ้ายมือ)
 
Word2vec สนับสนุนการเรียนรู้ด้วยเทคนิคจำนวน 2 เทคนิค คือ
1. Continuous bag-of-word model 
2. Continuous skip-gram model  

Word2vec techniques

CBOW, Skip-gram model
ทั้งสองโมเดลเป็นวิธีเรียนรู้เครื่องจักรเพื่อทำนายคำ โดยอาศัยคอร์ปัสซึ่งเป็น training data ช่วยในการวิเคราะห์ความน่าจะเป็นในการทำนายคำเหล่านั้น

Continuous bag-of-word model 
เป็นแบบจำลองเพื่อทำนายคำที่เป็นไปได้เมื่อกำหนดคำที่เป็นบริบทแวดล้อม
เช่น  Have.....great  day
เช่น หากกำหนด target คือ a คำที่เป็นบริบทแวดล้อมคือ have great และ day จะถูกนำไปวิเคราะห์ความน่าจะเป็นที่จะได้ target คือคำว่า a 

Continuous skip-gram model
เป็นการทำนายคำที่จะอยู่ก่อนหรือหลังคำที่กำหนด สมมติใช้ประโยคตัวอย่างแบบเดียวกันกับตัวอย่างข้างต้น
เช่น  ..... a .......  ...... 
ภาพตัวอย่างการทำงานของการเรียนรู้เครื่องจักรใน CBOW และ Skip-gram model

Word2vec ด้วย Gensim

Import Word2vec จาก Gensim

from gensim.models import Word2Vec

กำหนดชุดข้อมูลเทรน


sentences = [['compiler', 'interpreter', 'implementation'],
['compilation', 'compiler', 'machine code', 'compilation', 'compiler'],
['Java', 'is', 'compilation'],
['C', 'is', 'compilation'],
['Pascal', 'is', 'compilation', 'program']]

คำสั่งเทรนโมเดลด้วย CBOW technique

model = Word2Vec(sentences, size=100, min_count=1,sg=0)

คำสั่งเพื่อทำงานกับโมเดล

# แสดงสรุปเกี่ยวกับโมเดล
print("Model summary")
print(model)

# แสดงคำศัพท์ทั้งหมด
word = list(model.wv.vocab)
print(word)

# แสดงค่าในเวคเตอร์ที่แสดงคำว่า compilation
print("Word embedding vector of specific word ")
print(model['compilation'])

# แสดงคำที่ใกล้เคียงจำนวนสามอันดับแรก()
result = model.most_similar(positive=['compilation','compiler','implementation'], topn=3)
print(result)
#เปรียบเทียบ Similarity ระหว่าง Compiler และ compilation
print(model.similarity('compiler','compilation'))

คำสั่งบันทึกและโหลดโมเดล

# คำสั่งในการบันทึกโมเดลเพื่อนำไปใช้ต่อโดยไม่ต้องรันใหม่
model.save('model.bin')
# คำสั่งในการโหลดโมเดลที่บันทึกไว้มาใช้งาน
new_model = Word2Vec.load('model.bin')
print(new_model)
